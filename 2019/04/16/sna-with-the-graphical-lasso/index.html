SNA with the Graphical Lasso

2019-04-16

Imagine you’re a financial advisor that has access to transactional data from the accounts of several thousand individuals, for example, one year of daily transactions from across 100k people who have been investing in the stock market. You have a finite budget for marketing your services and would like to expend the least amount possible on marketing while effectively reaching the greatest number of potential clients. A tool that reveals the social network structure of potential clients is incredibly useful for maximizing the efficiency of a marketing campaign, allowing you to understand the structure of who is influencing who, where the hubs of the network are (influential nodes with many connections), and how information flows between members of the social network.

In this blog post I will be coding a tool that reveals the social network structure from transactional data, requiring only a unique identifier for each individual, and a timestamp and quantity for each transaction. We will generate an undirected graph from these 3 pieces of information, with each vertex representing a person in the network. The absence of an edge between two vertices means that the transactional behavior of the corresponding individuals is conditionally independent.

The graphical lasso is a method that generates a large sparse matrix with edges representing conditional dependence between nodes in the network. Although an R package (glasso) already exists the graphical lasso function in the glasso package is not scalable and applying this method with high volume data becomes intractable as the amount of data grows larger. The version of the graphical lasso that is coded below is very fast, efficient, and scalable; making it a practical choice for analyzing high volume data.

In addition to coding a function that performs the graphical lasso we will be simulating transactional data, allowing us to evaluate how well the graphical lasso performs and is able to identify conditional dependence between the transactors in our sample.

First, we are going to simulate our transactional data. To simulate data we are going to use the “huge” package. The function “huge.generator” is used to simulate “n” transactions per “d” individuals. The sample being simulated condains 1000 individuals who have transactions across 1 year (365 days) and the social network structure is a set of several clusters of individuals.

require(huge)
## Loading required package: huge
d <- 1000
simdata = huge.generator(n=365,d=d,graph="cluster") # d individuals, 365 time points

#dependence structure between individuals is examined
simdata.in <- simdata$data
colnames(simdata.in) <- as.character(1:ncol(simdata.in))

Next, we will code our function that estimates the scaled partial correlations between each of the individuals from our sample. The pieces of the function are fairly simple, ensuring minimal copying and movement of data (using pointers), and applying lasso to perform sparse regression of each individuals transactions on every other (target) individuals transactions, while controlling for the effects of other (non-target) individual’s effects.

# Function to calculate an nxn sparse matrix containing sum of partial correlations estimated using lasso:
spc.matrix.uc <- function(indata,verbose=TRUE){
  scaled.data <- apply(indata,2,scale)
  require(glmnet)
  require(Matrix)
  require(igraph)
  
  ## Define pointer contstructor:
  
  newPointer=function(inputValue){  
    object=new.env(parent=globalenv())  
    object$value=inputValue  
    class(object)='pointer'
    
    return(object)  
  }  
  
  ## generate pointer to scaled.data:
  
  scaled.data.ptr <- newPointer(scaled.data)
  
  ## Perform computations
  
  labels <- colnames(scaled.data)
  grid <- 10^seq(10,-2,length=10) # for lambda selection
  
  # For first column of matrix:
  yindex <- 1
  
  # Find best lambda:
  
  cv.out <- cv.glmnet(scaled.data.ptr$value[,-yindex],scaled.data.ptr$value[,yindex],alpha=1)
  bestlam <- cv.out$lambda.min
  
  # Run lasso and only keep coefficients from best lambda
  lasso.mod <- glmnet(scaled.data.ptr$value[,-yindex],scaled.data.ptr$value[,yindex],alpha=1,lambda=bestlam,standardize=FALSE,nlambda = 1) # set alpha equal to 1 for lasso
  tmp_coeffs <- round(coef(lasso.mod),3)
  fillmat <- matrix(rep(0,length(labels),ncol=1))
  fillmat[yindex,1] <- -1
  fillmat[-yindex,1] <- tmp_coeffs[2:nrow(tmp_coeffs)] # exclude intercept (which is approx zero)
  spc.mat <- sparseMatrix(i=ncol(scaled.data),j=ncol(scaled.data),dims = c(ncol(scaled.data),ncol(scaled.data)),dimnames = list(labels,labels))
  spc.mat <- as(spc.mat, "dgCMatrix") # convert to sparse matrix that can store numeric values
  spc.mat[,1] <- fillmat
  
  
  for(i in 2:ncol(scaled.data)){
    yindex <- i
    # Run lasso and only keep coefficients from best lambda
    lasso.mod <- glmnet(scaled.data.ptr$value[,-yindex],scaled.data.ptr$value[,yindex],alpha=1,lambda=bestlam,standardize=FALSE,nlambda = 1) # set alpha equal to 1 for lasso
    tmp_coeffs <- round(coef(lasso.mod),3)
    fillmat <- matrix(rep(0,length(labels),ncol=1))
    fillmat[yindex,1] <- -1
    fillmat[-yindex,1] <- tmp_coeffs[2:nrow(tmp_coeffs)] # exclude intercept (which is approx zero)
    spc.mat[,i] <- fillmat
    if(verbose==TRUE){
      print(paste("estimation iteration ",i,sep=""))
    }
  } # end first loop i
  
  
  # use symmetry for error reduction:
  for(i in 1:ncol(scaled.data)){
    # If zero is detected in row or column fill in zero for both rows and columns (non-zeros are small and due to rounding error)
    rz.index <- spc.mat[i,]==0
    cz.index <- spc.mat[,i]==0
    spc.mat[rz.index,i] <- 0 
    spc.mat[i,cz.index] <- 0 
    # assign average value of spc.mat[i,j] and spc.mat[j,i] to remove asymmetry due to rounding error
    rv.index <- spc.mat[i,]>0
    cv.index <- spc.mat[,i]>0
    temp <- cbind(spc.mat[rv.index,i],spc.mat[i,cv.index])
    if(nrow(temp) >0){
      temp <- apply(temp,1,mean)
      spc.mat[rv.index,i] <- temp
      spc.mat[i,cv.index] <- temp
    }
    if(verbose==TRUE){
      print(paste("parameter balancing iteration ",i,sep=""))
    }
  } # end second loop i
  return(spc.mat)
}

# Compile R function spc.matrix.uc to spc.matrix
require(compiler)
## Loading required package: compiler
spc.matrix <- cmpfun(spc.matrix.uc)

The following code generates the matrix of coefficients. Although many of the generated coefficients indicate true dependence structure between individuals, some of the coefficients are false positives. The code below also removes edges that are below a threshold of a little over one standard deviation above and below the median coefficient value to reduce the likelihood of including false positives. The first image produced by the code below shows the raw coefficient matrix, while the second shows our histogram used for thresholding, and finally we see the proximity matrix of the graph. In both the matrix of coefficients and the graph the cluster structure of our network is clearly visible along the diagonal.

graph.nodes <- cbind.data.frame(graph.nodes,rep(1,nrow(graph.nodes)))
